# PRD: Project MERCENARY
### Area 88 영감 가로형 횡스크롤 슈팅 게임

---

## 1. 개요

### 1.1 제품 비전
Capcom의 Area 88(1989)을 오마주한 PC용 횡스크롤 슈팅 게임.
단순한 슈팅이 아닌 **용병 경제 시스템**과 **기체/무기 커스터마이징**이 결합된 전략적 액션 게임.

### 1.2 핵심 컨셉
- 플레이어는 용병 파일럿으로 전쟁터에 투입됨
- 미션마다 적을 격추해 자금을 획득, 기체·무기를 업그레이드
- 스테이지 클리어보다 **생존과 자원 관리**가 핵심

### 1.3 타겟 유저
- 레트로 슈팅 게임 팬 (1980~90년대 아케이드 경험자)
- 로그라이트/전략 요소를 좋아하는 코어 게이머
- 연령대: 25~45세, PC 게이머

### 1.4 플랫폼
- **Primary**: PC (Windows)
- **Engine**: 미정 (Godot 4 또는 Unity 권장)
- **해상도**: 1920×1080 기준, 16:9 와이드
- **프레임레이트**: 60fps 고정

## 2. 핵심 기능 요구사항

### 2.1 파일럿 시스템
| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 파일럿 선택 | 3인 파일럿, 각기 다른 초기 기체·능력치 | P0 |
| 파일럿 스탯 | 공격력, 방어력, 기동성, 특수무기 슬롯 수 | P0 |
| 파일럿 스토리 | 각 파일럿별 배경 스토리 및 엔딩 분기 | P1 |

**파일럿 설계 (초안)**
- **신 (Shin)**: 균형형 — 기동성·공격 밸런스, 초보자 권장
- **믹키 (Mickey)**: 공격형 — 화력 최고, 방어 취약
- **그렉 (Greg)**: 방어형 — 체력 높음, 속도 느림

### 2.2 경제 시스템 (핵심 차별점)
| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 자금 획득 | 적 격추 시 달러($) 획득, 보스 격추 시 보너스 | P0 |
| 격추 보너스 | 연속 격추 콤보 시 자금 배율 증가 | P1 |
| 미션 완료 보너스 | 잔여 체력·시간에 따른 추가 자금 | P1 |
| 부채 시스템 | 자금 부족 시 대출 가능, 이자 발생 | P2 |

### 2.3 기체 시스템
| 기체 등급 | 예시 기종 | 가격 | 특징 |
|----------|----------|------|------|
| 기본 | F-8 Crusader | 무료 | 초기 지급 |
| 중급 | F-20 Tigershark | $50,000 | 속도·화력 향상 |
| 고급 | F-14 Tomcat | $120,000 | 특수무기 슬롯 추가 |
| 최고급 | F-15 Eagle | $200,000 | 최강 스펙 |

- 기체 파괴 시 해당 기체 소멸 (영구 손실)
- 기체 보험 구매 가능 (P2)

### 2.4 무기 시스템
**기본 무기** (무제한)
- 기관포 (Vulcan) — 전방 직선 공격

**구매 무기** (스톡 제한)
| 무기 | 효과 | 가격 |
|------|------|------|
| 공대공 미사일 | 적 추적, 단일 대상 | $3,000 |
| 확산탄 | 전방 부채꼴 범위 | $2,000 |
| 레이저 | 관통, 고데미지 | $5,000 |
| 폭탄 | 지상 적 특화, 범위 피해 | $4,000 |
| 핵미사일 | 전체 화면 클리어 (1발) | $30,000 |

### 2.5 미션 구조
- **총 스테이지**: 8개 스테이지 × 3~4 웨이브
- **미션 브리핑**: 스테이지 시작 전 목표 제시
- **중간 보급**: 스테이지 중 보급기 등장, 비행 중 무기 구매 가능
- **보스전**: 각 스테이지 말미 대형 보스 등장

## 3. 게임플레이 메카닉

### 3.1 이동 및 조작
- 8방향 자유 이동 (키보드: WASD / 방향키)
- 스크롤 속도: 일정하게 자동 진행, 플레이어가 앞뒤로 조정 가능 (±20%)
- 히트박스: 기체 중심부 소형 판정 (관대한 판정으로 스트레스 최소화)

### 3.2 피격 시스템
- 체력 게이지 방식 (HP Bar)
- 피격 시 무적 프레임 0.5초
- 기체 파괴 시 → 보유 기체 중 다음 기체로 교체
- 모든 기체 파괴 = 게임오버

### 3.3 적 구성
| 적 유형 | 행동 패턴 |
|--------|---------|
| 전투기 | 편대 비행, 직선 돌격 |
| 폭격기 | 느린 속도, 고체력, 폭탄 투하 |
| 헬기 | 상하 회피 기동 |
| 지상 포대 | 고정형, 포물선 포격 |
| 보스 | 다단계 패턴, 부위 파괴 |

### 3.4 보스 설계 원칙
- 최소 3단계 패턴 변화
- 부위 파괴 시스템 (날개, 포탑 등 개별 파괴 가능)
- 파괴 부위에 따라 공격 패턴 변화
- 격파 후 대량 자금 지급

|--------|------|------|---------|

| 6 | 해저 기지 | 잠수함 | 특수 지형 |
| 7 | 적 본부 | 다형 보스 | 고난이도 |
| 8 | 최종 결전 | 최종 보스 3단계 | 엔딩 분기 |

## 5. UI/UX 요구사항

### 5.1 HUD (게임 중 화면)
- 좌상단: HP 게이지, 기체 아이콘 (보유 기체 목록)
- 우상단: 현재 자금($), 선택 무기 및 잔탄
- 하단: 무기 슬롯 (1~4번 슬롯, 단축키 전환)

### 5.2 상점 화면 (미션 사이)
- 기체 구매/비교 화면
- 무기 구매 (잔탄 단위 구매)
- 현재 자금 표시
- "출격" 버튼으로 미션 시작

### 5.3 게임오버 화면
- 총 격추 수, 획득 자금, 생존 스테이지 표시
- 처음부터 / 마지막 체크포인트부터 선택

## 7. 비기능 요구사항

- **접근성**: 색맹 모드, 화면 흔들림 On/Off
- **난이도**: Easy / Normal / Hard / Arcade (목숨 제한) 4단계
- **성능**: 적 50기 동시 등장 시에도 60fps 유지
- **세이브**: 스테이지 단위 자동 저장, 슬롯 3개
